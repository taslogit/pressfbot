PRESS F

Это проект про цифровые тайники, споры и выживание в стиле Telegram WebApp. Я делал его один, от идеи в голове до рабочей версии: постепенно наращивал функционал, ловил баги, чинил UX и докручивал стиль. Сейчас это уже не демка, а живой продукт со своей экосистемой.

Что внутри
Фронт: Vite + React.
Бэк: Node + Express.
Данные: Postgres.
Очереди: Redis + Bull.
Инфра: Docker + Traefik + Let’s Encrypt.

Как работает
Пользователь открывает WebApp из Telegram. Бэк валидирует initData, выдаёт sessionId, дальше все запросы идут через API. Контент хранится в Postgres, фоновые задачи улетают в очередь. Traefik проксирует /api и /bot и держит HTTPS.

Функционал по сути
Письма‑капсулы с таймерами и датами открытия, история версий и экспорт. Дуэли/бифы с публичной драмой, итогами и шерингом. Профиль, статистика, уведомления, настройки. TON‑часть для наследок, escrow и вечного хранения.

Безопасность
Проверка Telegram initData на сервере. Доступ к API по sessionId. SSL через Let’s Encrypt. Секреты в .env, в Git они не живут.

Telegram и бот
Бот встречает инфо‑блоком и кнопкой НАЧАТЬ, открывает WebApp, принимает апдейты через /bot вебхук. Если коротко: бот нужен, чтобы вход был быстрый и нативный.

Блокчейн и TON
TonConnect для привязки кошелька. Идея простая: часть логики фиксируется on‑chain, сами данные лежат off‑chain. Так легче доказать факт, но не палить контент.

Стиль и подача
Текст, инфо‑разделы и названия в мем‑тоне. Это фича, не баг. Если где‑то слишком душно, значит ещё не допилил.

Если хочешь понять идею за минуту: PRESS F — это твой цифровой сейф и арена для разборок, завязанная на Telegram и на живую культуру мемов.
